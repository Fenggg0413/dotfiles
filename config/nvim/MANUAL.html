1. 概览

基于 LazyVim 模版，所有插件版本锁定在 lazy-lock.json

关键文件

init.lua入口，仅 require("config.lazy")
lua/config/lazy.lua引导 lazy.nvim + LazyVim spec
lua/config/options.luavim option 覆盖（追加于 LazyVim 默认之后）
lua/config/keymaps.lua额外 keymap
lua/config/autocmds.lua额外 autocmd
lua/plugins/colorscheme.lua主题 = tokyonight-night
lazyvim.json启用的 extras 清单 + 已读 NEWS 版本
lazy-lock.json插件 commit 锁文件（跟仓库）
.neoconf.json项目级 LSP overrides（neoconf）

2. 自定义改动（区别于 LazyVim 默认）

来源改动效果
keymaps.lua <C-d> / <C-u> 重映射为 <C-d>zz / <C-u>zz 半页翻动后自动 zz 居中光标，减少视觉跳变
options.lua conceallevel = 0 markdown 不再隐藏 */_/链接括号；render-markdown 仍可读
autocmds.lua FileType markdown/gitcommit/text 内 spell = false LazyVim 默认开 spell，会把 tmux、zsh、fzf 等 CLI 名标错。整体关闭，保留 wrap

4. 文本对象 / Surround

内建 + mini.ai 文本对象

{operator}{a/i}{obj}，例如 diw、ca"、vi)。i=inner（不含边界），a=around（含边界）。

对象含义
w W词 / 大词
s p句子 / 段落
" ' `引号包围
( ) b圆括号
[ ]方括号
{ } B花括号
< > t尖括号 / HTML 标签
f c函数调用 / class（mini.ai 扩展）
o代码块（mini.ai：if/for/while…）
?用户输入分隔符（mini.ai）

treesitter-textobjects 额外提供 af/if（函数）、ac/ic（class）等基于 AST 的对象。

mini.surround（来自 extra）

前缀统一为 gs，避免与 s（Flash）冲突。

键动作
gsa{motion}{c}给目标加包裹符 {c}
gsd{c}删除最近的 {c} 包裹
gsr{from}{to}替换包裹符
gsf{c} gsF{c}跳到右/左侧的 {c}
gsh{c}高亮匹配
gsn调整搜索范围行数

示例：gsaiw" 给当前词加双引号；gsr"' 把双引号换成单引号；gsd( 去掉最近一对圆括号。

treesitter-context

顶部固定显示当前光标位置的父级（函数 / class / if 等），随滚动更新；纯展示，无按键。<leader>ut 可切换 treesitter 总开关。

5. Flash 跳转

输入字符后屏幕上所有匹配位置出现 1-2 字母标签，敲标签直达。可视/操作模式可用。

键动作模式
sFlash 跳转n/x/o
SFlash treesitter（按 AST 节点跳转/选择）n/x/o
rRemote Flash（先跳，做完操作返回原位）o
Rtreesitter 搜索（围绕节点查找）o/x
<C-s>cmdline 内切换 Flash 搜索辅助c

小心：s 覆盖了原 Vim 的「替换字符并进入插入」。改用 cl 等价。

14. 插件功能简介

仅列出 lazy-lock.json 中实际安装的插件。版本通过 commit hash 锁定，:Lazy sync 后会更新该文件。

核心 / 引导

lazy.nvim

插件管理器。:Lazy 打开管理面板。

LazyVim

预设发行版本体，提供默认 keymaps / options / autocmds / 插件 spec。

plenary.nvim

Lua 工具库，多数插件依赖。

nui.nvim

UI 组件库（noice、trouble 等依赖）。

snacks.nvim

LazyVim 自研一站式工具：picker / explorer / terminal / dashboard / notifier / scratch / words / scope / scroll / statuscolumn / bigfile / quickfile / indent。

which-key.nvim

按下 <leader> 后等 200ms 弹出可用键位提示。

编辑 / 文本对象

mini.ai

扩展文本对象：af/if（函数）、ac/ic（class）、ao/io（代码块）。

mini.pairs

自动补全括号引号。

mini.surround

包裹符增删改，前缀 gs。

mini.icons

统一 nerd font 图标供应。

flash.nvim

2 字母标签跳转 + treesitter 节点选择。s / S。

ts-comments.nvim

treesitter 感知的注释（修正嵌入语言注释符）。gcc / gc。

nvim-ts-autotag

HTML/JSX 标签自动闭合 + 同步重命名。

todo-comments.nvim

TODO/FIX/HACK/NOTE/WARN 高亮 + 跳转。]t/[t、<leader>st。

UI / 显示

tokyonight.nvim

主题。当前用 tokyonight-night。<leader>uC 切换。

catppuccin

备用主题（已装未启用）。

bufferline.nvim

顶部 buffer 标签栏。

lualine.nvim

状态行。

noice.nvim

消息 / cmdline / popup 重做。:Noice 查历史。

nvim-treesitter

AST 高亮 / 缩进 / 折叠基础。:TSUpdate。

nvim-treesitter-context

顶部固定父级上下文。

nvim-treesitter-textobjects

基于 AST 的文本对象（与 mini.ai 协同）。

render-markdown.nvim

buffer 内 markdown 富渲染（标题样式、列表标记等）。

LSP / 补全 / 格式化

nvim-lspconfig

官方 LSP 客户端配置集合。

mason.nvim

LSP / DAP / linter / formatter 包管理器。:Mason。

mason-lspconfig.nvim

Mason 与 lspconfig 之间的桥接。

blink.cmp

补全引擎。<Tab>/<S-Tab> 选择，<CR> 确认，<C-Space> 触发，<C-e> 关闭。

friendly-snippets

预置 snippet 集合，配合 blink.cmp。

conform.nvim

格式化派发器。:ConformInfo。

nvim-lint

异步 linter 派发器（按 FileType 触发）。

lazydev.nvim

编辑 Neovim 配置时为 Lua LSP 注入 nvim API 类型。

aerial.nvim

代码符号大纲侧栏。<leader>cs。

SchemaStore.nvim

提供 JSON/YAML schema 索引给 jsonls/yamlls。

Git / 诊断 / 搜索替换

gitsigns.nvim

侧栏 hunk 指示 + stage/reset/preview/blame。

trouble.nvim

诊断 / quickfix / LSP 引用 美化列表。<leader>x*。

grug-far.nvim

项目级查找替换 UI。<leader>sr 或 :GrugFar。

语言专用

venv-selector.nvim

Python 虚拟环境选择。<leader>cv。

markdown-preview.nvim

会话
